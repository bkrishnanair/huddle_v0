Your task is to allow users to enter and explore the application without creating an account. Guests should be able to view the Map and Discover pages. They should be prompted to sign up when they attempt to access protected features like My Events or Profile.

Architectural Implementation Plan (Non-Destructive):

You will implement this feature by following these five steps:

Introduce a Global "Guest Mode":

File: lib/firebase-context.tsx
Action: Update the FirebaseContext to manage a new isGuest boolean state. This state should be persisted in the browser's sessionStorage to ensure it is instantly available across page loads and redirects, preventing UI flickering. Create enterGuestMode and exitGuestMode functions that manage this state and the sessionStorage item.
Create the Guest Entry Point:

File: components/auth-screen.tsx
Action: Add a "Continue as Guest" button to the authentication screen. When clicked, this button should call the enterGuestMode function from the context.
Update the Authentication Gateway:

File: app/(app)/layout.tsx
Action: Modify the layout's protective logic. It should grant access if a user is authenticated (user object exists) OR if they are in "Guest Mode" (isGuest is true).
Create a Reusable "Guest Prompt":

Action: Create a new component, components/guest-prompt.tsx.
Functionality: This component will be displayed on pages that are inaccessible to guests. It should show a title, a message, and a clear "Login or Create Account" button that calls exitGuestMode and redirects the user to the landing page to sign in.
Conditionally Adapt the UI:

File: components/bottom-navigation.tsx
Action: Modify the navigation bar to be "guest-aware." If isGuest is true, it should display a "Login" button (with a LogIn icon) in place of the "Profile" button.
